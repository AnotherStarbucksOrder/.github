## 💻 컨벤션

✔ 컨벤션
팀원 간의 원활한 소통과 협업을 위해 커밋 컨벤션과, 코드 컨벤션을 만들어 이를 따랐습니다. 
리드미에는 간략히 작성하고, 자세한 컨벤션은 각각의 타이틀에 링크된 깃허브 위키에 적어두었습니다.

#### 👀 커밋컨벤션
```
백엔드 커밋 컨벤션 

1. 커밋은 자신이 어떤 작업을 했는지 적고 뒤에 이름을 적는다.
2. 제목은 어떤 작업을 할 건지 다른 사람도 파악할 수 있도록 하기
3. 자신의 코드가 직관적으로 바로 파악할 수 있다고 생각하지 말자
4. 제목은 어떤 작업을 할 건지 다른 사람도 파악할 수 있도록 하기

    - FEATURE : 새로운 기능 추가
    - FIX: 버그 수정
    - UPDATE: 수정 사항 
    - TEST : 테스트 코드, 리팩토링 테스트 코드 추가

프론트 커밋 컨벤션 

1. 커밋은 자신이 어떤 작업을 했는지 적고 뒤에 이름을 적는다.
2. 제목은 어떤 작업을 할 건지 다른 사람도 파악할 수 있도록 하기
3. 자신의 코드가 직관적으로 바로 파악할 수 있다고 생각하지 말자
4. 제목은 어떤 작업을 할 건지 다른 사람도 파악할 수 있도록 하기

    - FEATURE : 새로운 기능 추가
    - UPDATE: 수정 사항
```

#### 👀 코드컨벤션
```
📝 문자열을 처리할 때는 쌍따옴표를 사용하도록 합니다.

🔚 문장이 종료될 때는 세미콜론(;)을 붙여줍니다.

🔄 반복문은 map, forEach, for, filter와 같은 배열 메서드를 권장합니다.

🛠️ 메서드 이름은 동사로 시작하며, 수행하는 작업을 명확히 나타내도록 작성합니다.

🔍 문자열에 변수를 포함할 경우, 템플릿 문자열(`)을 사용하여 코드의 가독성을 개선합니다.

🔒 상수 및 변수 선언에서 const는 재할당이 필요없는 값에 사용하고, 변경이 예상되는 값에만 let을 사용합니다.

🐪 함수명, 변수명은 카멜케이스로 작성합니다.

📏 가독성을 위해 한 줄에 하나의 문장만 작성합니다.

🗒️ 주석은 설명하려는 코드 구문과 동일한 들여쓰기를 유지하여 가독성을 높입니다.

💬 함수를 정의할 때, 함수의 기능을 설명하려는 주석을 추가하여 유지보수를 용이하게 합니다.

🔠 상수에는 UPPER_SNAKE_CASE로 작성합니다.

♻️ 중복 코드가 많아지면 컴포넌트로 분리하여 코드의 재사용성을 높입니다.

➕ 연산자 사이에는 공백을 추가하여 가독성을 높입니다.

🧩 콤마 다음에 값이 올 경우 공백을 추가하여 가독성을 높입니다.

🛑 단일 태그일 경우 self-closing()을 사용하며 여러 줄에 걸쳐 작성될 경우 일관된 들여쓰기를 유지합니다.

❓ 삼항 연산자를 사용한 조건부 렌더링은 단일 조건일 때 사용하고, 여러 조건이 있는 경우 If문이나 && 연산자를 사용하여 가독성을 높입니다.

🎨 스타일 객체의 키는 카멜케이스로 작성합니다.

📁 파일명은 파스칼표기법을 사용하고, 컴포넌트 파일과 스타일 파일은 같은 폴더에 정리하여 구성 요소별 관리가 쉽도록 합니다.

📚 객체 및 배열 형식은 여러 개의 프로퍼티가 포함된 객체는 각 프로퍼티마다 한 줄씩 작성하여 멀티라인 형식을 유지합니다.

🗂️ 패키지 구조는 기능별로 명확하게 분리하여 계층화합니다 (ex. Controller, Service, Repository 등).

🏛️ 클래스 이름은 파스칼표기법으로 시작하는 형식으로 사용하고, 명확하고 직관적인 이름을 부여합니다.

📌 상수는 대문자와 언더스코어로 시작하고, 모든 상수는 클래스 상단에 위치시킵니다.

✉️ DTO는 요청 및 응답 처리에 역할을 명확하게 분리합니다.

🚨 커스텀 예외 클래스를 생성하여 의미 있는 예외 메시지를 전달하고, 글로벌 예외 처리를 통해 공통 예외 로직을 관리합니다.

🔙 리턴 값을 명확하게 합니다.

🌐 컨트롤러의 각 메서드에 의미 있는 HTTP 메서드를 사용하고, 엔드포인트는 소문자와 복수형으로 정의합니다.
```
